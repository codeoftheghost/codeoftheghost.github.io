Membuat dan Menjalanakan Kode Program Python#

Terdapat berbagai cara atau metode untuk menjalankan kode program python yang kita tulis. Diantaranya yang paling umum digunakan ialah menggunakan metode Interpreter Interaktif dan Python Script. Kedua metode ini, masing-masing memiliki kelebihan dan kekuranganya masing-masing. Berikut penjelasan dari kedua metode tersebut.

Interpreter Interaktif#

Interperet Interaktif atau ini dikenal juga dengan sebutan REPL (Read-Eval-Print-Loop) adalah metode untuk menjalankan program python, dimana kita menggunakan interpreter python dengan mode interaktif. Pada mode interaktif ini, kita dapat menulis dan menjalankan kode program python secara langsung, baris per-baris, dan melihat langsung hasilnya saat kita menekan tombol Enter.

Atau jika distribusi linux yang kalian gunakan masih menyertakan versi Python2 atau tidak menjadikan versi Python3 sebagai default, maka jalankan perintah python3 seperti contoh dibawah ini:


python

Pada gambar diatas saat kita menjalankan perintah python, interpreter python akan dijalankan dengan mode interaktif. Kemudian muncul teks berupa versi python yang digunakan berserta informasi lainnya dan juga simbol >>> yang disebut dengan prompt. Simbol prompt ini menunjukkan bahwa interpreter python siap menerima input/masukan berupa kode program python dari pengguna. Contoh saya akan memasukkan kode program print(“Hello World!”), saat saya menekan Enter, interpreter python akan langsung menjalankan kode program yang saya masukkan dan akan langsung menampilkan hasilnya berupa teks Hello World!.

Perhatikan hasil yang ditampilkan pada gambar diatas, tidak terdapat simbol >>>. Ini cara interpreter interaktif python memberitahukan kepada pengguna, mana perintah yang diinput/dimasukkan pengguna, dan mana hasil yang ditampilkan.

Untuk keluar dari interpreter interaktif ini, terdapat beberapa cara. Namun cara yang paling mudah ialah, kita bisa menggunakan kombinasi CTRL + D di keyboard.

Python Script#

Python script merupakan metode atau cara menjalankan kode program python, dengan cara menuliskan semua kode program kedalam sebuah file. File yang berisi kode program ini umumnya disebut sebagai file kode sumber/source code. Khusus pada bahasa pemrograman interpreted seperti pyton, file ini kadang juga disebut sebagai file script. Di catatan ini, saya akan menggunakan istilah file kode sumber untuk penyebutan file ini. Di python, file kode sumber memiliki ekstensi .py.

Untuk mulai menggunakan metode ini, pertama-tama kita buat sebuah file kode sumber. Pada contoh ini, saya membuat file kode sumber dengan nama hello_word.py.

Kalian bebas mau menamai file tersebut apa saja, tetapi harus menggunakan ekstensi .py. Karena extensi ini berfungsi sebagai identitas yang menunjukkan bahwa file tersebut ialah file kode sumber yang ditulis dengan bahasa pemrograman pyton. Selain itu, meskipun bisa tetapi disarankan untuk tidak menggunakan karakter spasi pada nama file. Kita bisa mengganti penggunaan karakter spasi dengan karakter karakter dash (-) atau underscore (_).

Kemudian buka file kode sumber yang sudah kita buat tadi dengan kode editor VSCode atau kode editor yang kalian gunakan, lalu ketikkan kode program dibawah ini:


print("Hello World!")

Simpan file tersebut, bisa dengan menggunakan shortcut ctrl + S. Kemudian buka terminal dan arahkan ke direktori tempat kita menyimpan file kode sumber tersebut. Pada contoh ini, saya menyimpan file kode sumbernya di direktori “Ghost/python-cheat-sheet/basic/hello_world/”.


cd Ghost/python-cheat-sheet/basic/hello_world/

Kemudian jalankan file kode sumber tersebut dengan perintah python <nama-file-kode-sumber.py>, seperti contoh dibawah ini:


python hello_world.py

Setelah kita menekan Enter, interpreter python akan menjalankan kode program yang kita tulis pada file kode sumber hello_world.py, dan pada terminal akan tampil teks “Hello World!”.

Kelebihan dan Kelemahan dari Masing-masing Metode#

Dari dua metode yang sudah saya jelaskan diatas secara ringkas, masing-masing metode memiliki kelebihan dan kelemahannya masing-masing. Berikut beberapa diantaranya:

Kelebihan Metode Python Script#

Metode ini memiliki beberapa kelebihan untuk kita gunakan dalam menjalan kode program python. Diantaranya ialah:

  • Cocok untuk mengembangkan program yang lebih besar dengan banyak file kode sumber. Kode program dapat kita bagi kedalam fungsi, kelas, dan modul, serta dapat kita pisahkan menjadi beberapa file kode sumber agar kode program yang kita tulis menjadi lebih terstruktur.

  • Kode yang kita tulis bersifat persistent, karena tersimpan didalam file kode sumber. Sehingga, kita dapat dengan mudah membagikan kode yang kita tulis dengan orang lain, dan dapat kita jalankan kapan saja tanpa harus menulis kembali kode programnya.

Kekurangan Metode Python Script#

Tentu selain memiliki kelebihan, metode ini juga memiliki kekurangannya. Diantaranya ialah:

  • Kode program yang kita tulis tidak dapat langsung kita lihat hasilnya seperti di metode interpreter interaktif. Karena metode ini mengharuskan kita membuat file kode sumber, menulis dan menyimpan kode program-nya, lalu menjalankannya untuk melihat hasilnya.

  • Karena kita tidak dapat melihat hasilnya secara langsung, metode ini menjadi tidak ideal untuk eksperimen cepat. Dimana jika kita ingin mencoba kode program python tertentu dan ingin langsung melihat hasilnya, menggunakan metode ini agak berlebihan dan merepotkan.

Kelebihan Metode Interpreter Interaktif#

  • Metode ini sangat cocok untuk digunakan bereksperimen cepat seperti mencoba ide atau menguji konsep. Karena kita dapat dengan mudah menulis, menjalankan, dan dapat langsung melihat hasilnya.

  • Bagi pemula yang baru mempelajari bahasa pemrograman python, metode ini sangat berguna untuk memudahkan pemahaman kita tentang konsep dan fungsi yang disediakan oleh bahasa python.

  • Tidak membutuhkan persiapan lebih lanjut, karena kita cukup membuka interpreter python dengan terminal, dan kita sudah langsung bisa melakukan pemrograman dengan python.

  • Sangat membantu dan efektif untuk debugging, karena kita bisa menjalankan perintah satu per-satu dan melihat hasilnya. Ini memudahkan kita dalam menemukan dan memperbaiki bug.
